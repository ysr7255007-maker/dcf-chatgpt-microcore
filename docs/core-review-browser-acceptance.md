# DCF Core Review 浏览器验收单

## 隔离

验收时只启用 Core Review，暂时停用完整捆绑审查版。完整捆绑版保留作为回滚基线。

## A. 空 Core

1. 安装 Core Review userscript。
2. 打开新的 ChatGPT 页面。
3. 左下角应出现 Core Review 入口。
4. 展开后应显示当前快照为空、已安装包为空、已存模块为 0。

## B. 导入官方插件工具包

1. 使用面板内默认地址导入官方包。
2. 面板应显示官方包已安装，代码单元数量大于 0。
3. 刷新后，已安装包与代码单元仍应存在。

## C. 加载 minimal

1. 点击“加载 minimal”。
2. 页面刷新后应出现 DCF Shell。
3. 应启动页面交互、插件管理器和维护诊断。
4. Core Review 面板中 boot 应为 completed，最近可用快照应等于当前快照。

## D. 已安装与当前运行分离

1. 打开插件面板。
2. 将语言弹药、外观和备份加入当前组合并重启。
3. 刷新后这些插件应启动；未加入的插件不应自动运行。

## E. 插件运行环境

1. 修改外观后刷新，确认设置保留。
2. 新建测试弹药后刷新，确认内容保留。
3. 维护诊断中的已启动插件和页面状态应与当前组合一致。

## F. 本机 Agent 插件

1. 通过插件面板把本机 Agent 加入当前组合。
2. 本机服务未运行时，插件应显示未连接，但 Core 不应进入恢复。
3. 完成一次 Echo 往返，确认结果返回原页面。

## G. 恢复

基础步骤通过后，再使用专门的故障插件版本验证启动失败、最近可用快照与最低恢复入口。

每一步记录页面现象、面板状态、控制台错误、当前快照 ID 和 boot.plugins。不能以“看起来正常”代替状态证据。
