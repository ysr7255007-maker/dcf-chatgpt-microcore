# ADR: DCF 工作区标签与选择式弹药界面

Date: 2026-07-17  
Status: accepted; live acceptance pending

## Context

随着第一方插件增加到九个，Shell 将每个面板无条件平铺成标签，导致窄宽度下后加入的功能不可见，也让高频工作与低频配置共享同一注意力层。语言弹药同时在每张卡片重复放置六个动作，列表越长，视觉噪声和误触成本越高。

DCF 的目标不是展示全部能力，而是降低用户在当前任务中的认知损耗。因此需要把“有哪些能力”与“当前工作区正在使用哪些能力”分开，并把“选择对象”与“对对象执行动作”分开。

## Decision

1. Shell 标签栏代表当前固定的工作区，不再代表全部已启用插件。
2. 初次迁移默认固定 `ammo` 与 `plugins`；若升级前存在其他当前活动面板，则保留该面板并固定一次，避免上下文突然消失。
3. `plugins` 即“功能”标签始终固定，不能从标签栏移除，也不能在功能页中停用。
4. 其他已启用面板由用户在“功能”页点击“添加到标签栏”或“移出标签栏”。移出标签栏不等于停用插件。
5. 未启用功能可以通过“启用并添加”一次完成，不要求用户理解两个分离步骤。
6. 标签栏始终提供左右箭头；箭头在固定标签之间循环切换。
7. 鼠标位于标签栏时，滚轮在固定标签之间快速切换；不要求精确点击窄标签。
8. 固定状态属于 Shell 插件数据，并通过页面自定义事件与功能管理插件协作；不新增底座协议或底座状态。
9. 语言弹药卡片只承担单选。每张卡片显示标题、用途、ID、标签与版本，不再重复动作按钮。
10. 弹药列表独立垂直滚动，当前选择保持明显高亮。
11. 所有常规动作集中到页面底部统一操作槽：
    - 库级动作：从当前对话提取、新建、GitHub 加载、复制便携库、导入文件；
    - 所选弹药动作：发射、插入、复制、更新、编辑、删除。
12. 发射与插入的既有语义保持不变；本轮只重构选择和操作布局，不重定义语言弹药协议。

## Rationale

这套结构把 DCF 的日常心智模型压缩为：

```text
在标签栏选择当前工作区
→ 在工作区选择对象
→ 在统一操作槽执行动作
```

低频能力仍然可发现，但不持续占用标签空间；同类动作只出现一次，用户不需要在重复按钮中重新定位。插件仍保持独立更新与独立失败边界，没有因此被合并进 Shell 或底座。

## Acceptance boundary

自动化测试可以证明固定状态、事件协议、箭头、滚轮、单选卡片和统一操作槽存在，但不能替代以下真实验收：

- 窄宽度下标签箭头与滚轮是否符合手感；
- 热更新和刷新后固定标签是否稳定；
- 功能页的添加、移除、启用并添加是否即时生效；
- 弹药卡片滚动、选择、编辑和底部操作是否在真实数据量下保持清晰；
- 从旧 Shell 状态迁移时是否保留用户当时正在看的面板。
