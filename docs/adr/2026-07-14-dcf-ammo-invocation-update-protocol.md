# ADR: DCF 语言弹药使用语境化调用与实质更新协议

Date: 2026-07-14
Status: accepted

## Context

语言弹药是从过去对话中沉淀出的高密度长期意图。旧发射机制只发送正文，接收窗口容易把高度凝练的历史表达误认为用户此刻逐字确认的明确命令，进而机械执行。旧更新请求只要求“根据当前对话更新”，没有明确区分文本润色、长期意图修订和自动装填协议。

## Decision

1. 发射消息使用 `〔DCF·语言弹药〕` 加一行空白和原始正文。标志声明正文需要结合当前项目和对话进行二次理解，而不是逐字机械执行。
2. 二次理解是必需的，二次确认是条件性的：当前落地清楚时适配后直接执行；关键含义冲突或存在无法可靠判断的多种落地方式时才确认。
3. 更新消息使用 `〔DCF·弹药更新〕`，并携带完整原弹药、当前对话作为修订语境、保留与变更规则，以及返回完整同 `id` `DCF_AMMO` 工件的输出协议。
4. 更新只吸收已经稳定形成、真正改变核心意图、适用范围、使用方式或关键边界的认识；不只做措辞润色，也不把当前对话机械摘要进正文，不另建相似弹药。
5. 调用标志、更新标志和修订规则由 `dcf.standard.ammo` 的 `ammo_protocol` policy 拥有。bootstrap 从 registry policy 读取，只有缺失时使用等价安全回退。
6. 复制操作继续导出原始正文。调用包络只属于发射，避免破坏编辑、迁移和外部复用。

## Consequences

- 语言弹药从“可复用文本片段”提升为带调用语义的长期意图资源。
- 低摩擦仍然保留：语境清楚时不强制用户再次确认。
- 更新输出可以直接进入现有 `DCF_AMMO` 摄取与 Environment Reconciler，不新增第二套更新状态。
- 首次支持 policy 读取需要 bootstrap `0.14.0`；之后普通协议措辞变化只发布新的 ammo package revision。
