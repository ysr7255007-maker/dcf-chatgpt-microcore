# ADR: DCF Core Review 真实浏览器验收

Date: 2026-07-17  
Status: accepted for experiment

## Context

官方插件工具包的源码与交付边界已经成立，但本机代码库存储、精确运行快照、插件上下文、GM 能力、启动失败和最低恢复仍需在真实 Tampermonkey 中共同验证。脱离浏览器的单元测试不能代替这一证据。

## Decision

1. 新增独立 `DCF ChatGPT Next Core Review` userscript，不替换当前完整捆绑审查版。
2. Core Review 只负责插件包显式导入、代码单元持久化、SHA-256 校验、精确快照、顺序启动、最近可用快照和最低恢复界面。
3. 插件包必须先由用户显式导入并写入本机代码库，未经安装的远程内容不会进入运行快照。
4. 当前官方插件源码与 `start(context)` 契约保持不变，Core Review 只提供现有插件实际需要的窄兼容面。
5. 插件管理器升级为普通插件版本 `1.1.0`，可以将已安装但尚未进入当前组合的插件加入下一份快照。
6. 自动测试只证明数据结构、哈希、模块解析和快照编译边界；正式成立必须以真实浏览器现场证据为准。
7. 任一关键条件失败时保留完整捆绑版，记录根限制，不增加并行运行路径掩盖失败。

## Acceptance boundary

必须在真实页面证明：

- 空 Core 能显示最低恢复入口；
- 官方插件包可以导入并在刷新后保留；
- 最小快照可以启动 Shell、ChatGPT、插件管理器和诊断；
- 插件管理器可以把其他已安装插件加入组合；
- 插件私有存储、插件间 API 和页面适配可以正常使用；
- 哈希不一致和插件启动失败不会成为最近可用快照；
- 插件管理器失败时仍可使用最低恢复入口返回上一份快照；
- Local Agent 通过同一安装和快照链路运行。

## Consequences

- 运行机制的不确定性被隔离在独立实验脚本中，不影响当前可用捆绑版。
- 验收通过后，才能删除静态注册表和捆绑清单协调逻辑。
- 验收失败时，官方插件工具包仍然保留为有效的源码与发布边界。
