# ADR: DCF 跨平台记忆系统方向

Date: 2026-07-14  
Status: proposed

## Context

DCF 当前以 ChatGPT Tampermonkey Runtime 为主要宿主，`dcf.state.root.v1` 存放于浏览器 GM storage，并围绕语言弹药、能力包、环境政策、Profile、恢复和 Runtime 投影建立单一权威状态。这个结构已经能够维护当前 ChatGPT 对话环境，但用户的长期认知材料仍容易被浏览器、设备和具体 AI 平台切割。

语言弹药只是长期记忆中的一种高密度可复用语言。项目状态、决策及其变化、个人偏好、稳定约束、未决问题、阶段检查点和证据来源同样可能需要在 ChatGPT、Codex、Claude、Gemini、本地模型与未来平台之间持续可用。如果每个平台各自保存一份不可追踪的记忆，DCF 会重新落入多份状态、来源不明和同步冲突。

## Proposed direction

1. DCF 后续可以增加跨平台记忆能力，但它应被视为平台中立的个人认知基础设施，而不是当前油猴脚本中的并列记忆模块。
2. 不在每个平台建立可独立编辑的完整副本。未来应形成一个明确的 canonical memory store；各平台只通过适配器读取、提交变更或保留有界缓存。
3. 当前 `dcf.state.root.v1` 在架构迁移前继续作为 ChatGPT 期望环境的唯一权威状态。它可以保存记忆适配器配置、已采用记忆引用、同步游标、缓存来源和投影状态，但不得与外部 canonical store 同时成为同一记忆对象的双权威。
4. 跨平台记忆采用稳定对象协议。每条记忆至少具有稳定 ID、revision、scope、kind、provenance、时间、状态、敏感级别和生命周期信息；内容格式保持可读、可导出并允许确定性校验。
5. 记忆不会默认等于完整对话归档。平台适配器根据当前项目、任务和对话需求选择有限相关切片，再编译为 typed Artifact / Intent 或平台可消费的上下文投影；禁止把全部历史无差别注入每次对话。
6. 语言弹药、ADR、项目检查点和稳定偏好继续保留自身语义与使用协议，跨平台记忆层负责统一寻址、来源、版本、选择和投影，不把不同资源压平成无类型文本块。
7. 系统优先 local-first、用户可控和可迁移。任何远程同步必须显式启用，凭据与高敏内容不得进入普通回执、Catalog 或公开仓库；是否加密以及密钥托管方式需要单独决策。
8. 第一阶段先验证可移植记忆工件与一个 canonical store 的完整闭环：创建、修订、选择、投影、冲突检测、导出和恢复。向量数据库、自动全量提取、主动推送和多端实时协同不作为起步前提。

## Architectural consequence

跨平台记忆如果成立，DCF 的范围将从“ChatGPT 中的期望对话环境”向“个人跨平台认知环境”扩展。现有 userscript 应逐步成为 ChatGPT Adapter，而不是继续承担全部持久化、同步、检索和平台适配职责。

这会要求重新明确两类权威对象：

- 平台中立记忆对象由 canonical memory store 权威维护；
- 当前平台的环境选择、界面、能力包和投影状态仍由对应环境根维护。

二者通过稳定引用和事务化摄取连接，而不是通过双向复制同一份可编辑内容连接。

## Open questions

- canonical memory store 首版采用可读文件/Git、SQLite、本地服务还是可选远程同步；
- 语言弹药、ADR、项目状态与普通事实记忆的类型边界；
- 多端离线修改时的 revision 与冲突处理；
- 平台适配器通过浏览器扩展、MCP、本地服务、CLI 还是平台原生连接器接入；
- 记忆选择由显式调用、规则、检索模型还是混合机制完成；
- 删除、遗忘、过期、撤销和敏感信息隔离的可验证语义。

## Acceptance conditions

只有在下列条件满足后，本方向才应从 proposed 转为 accepted：

1. 已有至少两个真实平台需要共享同一类长期认知资源；
2. 已确定 canonical store 与单对象单权威原则；
3. 已证明不会破坏现有语言弹药低摩擦闭环；
4. 已定义最小记忆对象协议、冲突语义和隐私边界；
5. 已完成一个端到端原型，证明跨平台投影优于手工复制且不会把维护复杂度交还给用户。

## Reconsideration

如果实际需求只是把少量语言弹药从 ChatGPT 复制到其他平台，应优先增强可移植工件和导出入口，而不是建立完整记忆系统。只有当跨平台持续修订、选择、来源追踪和状态延续成为重复真实需求时，才升级为独立基础设施。
