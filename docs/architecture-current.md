# DCF 当前架构：Chrome 原生动态宿主候选

Updated: 2026-07-17  
Candidate: `1.0.0-rc.1`  
Legacy fallback: Tampermonkey `0.18.2`

## 1. 价值主权

DCF 的产品价值是低摩擦语言弹药闭环。Chrome 宿主、代码单元、快照、构建和更新机制只拥有工程主权。任何实现变化都必须先检查是否保留自动装填、同 ID 更新、可见版本、低摩擦发射、迁移和恢复，并且没有把内部复杂度交还给用户。

## 2. 单一 Chrome 宿主

用户只安装一个 Manifest V3 扩展。扩展只面向 Google Chrome 与 ChatGPT，不建设通用 userscript 管理器、跨浏览器适配层、GM API、云账户或本机守护进程。

静态扩展拥有：

- `chrome.storage.local` 中的代码库、精确快照、最近可用快照、启动证据与产品数据；
- `chrome.userScripts` 的注册、更新、注销和查询协调；
- 独立的首次启用页、迁移桥和最低恢复页；
- 对 USER_SCRIPT world 的窄消息协议。

## 3. 两个统一对象

### 3.1 受控可执行代码单元

每个单元包含稳定 ID、版本、构建后 JavaScript、SHA-256、来源、适用页面、执行时机、USER_SCRIPT world、宿主 API 版本、阶段和必要性。版本不可变；同 ID 新版本进入本地候选区，不覆盖历史代码。

### 3.2 精确启动快照

快照只引用代码单元的精确 ID、版本、哈希、启用状态和阶段。安装、更新、启停、运行、回滚、恢复与体检都通过代码库和快照解释，不再分别建设生命周期。

## 4. 候选确认与回滚

```text
代码进入本地库
→ 校验全部引用和哈希
→ 保存 candidate snapshot
→ 与 Chrome 实际注册集合协调
→ 等待每个启用单元的启动证据
→ 全部成功后提交 current + last_known_good
```

注册异常或代码单元报告启动失败时：

- candidate 不覆盖 last_known_good；
- 清理故障组合；
- 恢复 last_known_good；
- 保留候选代码和最小错误证据。

Chrome 扩展更新会清空动态用户脚本，因此 `runtime.onInstalled` 与 `runtime.onStartup` 都从 candidate/current/LKG 重新建立注册集合。

## 5. 最小生存核

生存核是静态 service worker、迁移 content script、onboarding 与 recovery 页面。它不理解对话语义，只理解代码单元、快照、哈希、注册、证据和窄产品数据存取。

恢复页不依赖任何动态代码单元，可以直接查看：

- 本地保存的代码版本；
- candidate/current/LKG；
- Chrome 实际注册脚本；
- 最近证据与偏差；
- 恢复 LKG、重建集合、停用故障单元和复制诊断包。

## 6. 语言弹药代码单元

`dcf.firstparty.ammo@1.0.0-rc.1` 自包含 ChatGPT 页面适配、语言弹药 UI 与闭环：

- 只观察新增助手回复，并对固定的最后三条做启动补偿；
- 解析完整 `DCF_AMMO` 工件；
- 同一 ID 原位更新并递增可见版本；
- 原子保存到扩展权威状态；
- 支持提取、新建、编辑、搜索、复制、语境化发射、更新、删除、导入和导出；
- 页面刷新、浏览器重启和扩展更新后从宿主恢复。

动态代码单元不直接获得 `chrome.storage`、tabs 或广泛权限，只通过 `chrome.runtime.onUserScriptMessage` 暴露的有限方法访问产品状态。

## 7. 旧版迁移

Tampermonkey GM storage 对 Chrome 扩展不可直接读取。候选不要求用户导出 JSON，而由静态迁移桥读取旧版开放 Shadow DOM 中已经呈现的真实数据：

- 每枚弹药的 ID、标题、用途、标签与完整正文；
- 当前发射模式；
- 可观察的侧栏方向与几何设置。

迁移以数量、唯一 ID 和内容哈希校验；冲突不静默覆盖，旧版数据始终保留。旧实现专属的包组合、Runtime 诊断缓存和内部操作记录不迁移，因为它们不是新产品状态。

## 8. 更新与来源

扩展内置官方代码单元用于首次安装。后续官方更新从固定 GitHub origin 读取索引和单元，逐项校验 ID、版本与 SHA-256 后进入同一候选快照链路。浏览器不解析 npm 依赖树，普通依赖由构建阶段打包。

## 9. 隐私

对话和语言弹药默认保存在浏览器本地。回复观察不扫描整页历史；诊断不包含消息正文、弹药正文、令牌或认证信息。远程请求只在用户检查官方更新时取得代码单元索引与代码，不上传本地内容。

## 10. 验证边界

自动验证覆盖：清单权限、哈希、代码库、快照、注册差异、启动证据确认、扩展更新后重建、失败候选回滚、同 ID 弹药更新、迁移桥、静态恢复面、确定性构建和唯一 ZIP。

真实 ChatGPT DOM 与 Chrome 的一次性用户脚本开关仍需唯一一次正常使用验收。候选未经过该验收前，不取代旧正式版。
