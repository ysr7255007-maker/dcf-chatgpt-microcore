# DCF 基本共识提示词

DCF 是用户与 AI 共同维护的个人认知基础设施。它的价值目标是低摩擦语言弹药闭环：从有价值的 ChatGPT 对话中形成高密度长期意图，自动装填、持续更新，并在后续语境中低摩擦发射。

语言弹药拥有价值主权，Chrome 宿主拥有工程主权。底座可以重写，产品目的不能被重写；内部减少的复杂度不能通过复制、粘贴、导入、确认、安装、版本切换或维护工作重新交给用户。

当前目标形态是一个只面向 Google Chrome 与 ChatGPT 的 Manifest V3 扩展。用户只安装这一个载体，不需要 Tampermonkey、本机 Runtime、Node.js、pnpm、Docker、命令行或常驻服务。旧 Tampermonkey 正式版仅保留为迁移期间回退。

统一工程对象是受控可执行代码单元和精确启动快照。安装是代码进入本地库，更新是同 ID 新版本进入候选，启停是候选快照变化，运行是宿主协调 Chrome 实际注册集合，回滚是恢复最近可用快照，体检是比较代码库、目标快照、实际注册和启动证据。

代码单元版本不可变，必须包含构建后 JavaScript 与 SHA-256。动态代码在 `USER_SCRIPT` world 执行，只通过窄协议访问宿主，不直接获得广泛扩展权限。浏览器不执行 TypeScript、不解析 npm 依赖树、不使用 eval、Blob 或 CSP 绕行。

候选只有在全部启用代码单元返回启动成功证据后，才能成为 current 与 last-known-good。注册或启动失败不得覆盖最近可用快照；必须清理故障组合、恢复旧集合并留下最小证据。扩展更新清除动态注册后，宿主从精确快照自动重建。

最小生存核只理解代码、哈希、快照、注册、证据、恢复和窄数据存取。它不理解语言弹药语义、对话意义、自动冲突判断或插件业务关系。高级管理能力损坏时，静态恢复页仍然可用。

语言弹药是第一方核心代码单元，不是平台示例。它只观察新完成的助手回复和固定恢复窗口；同一 ID 持续更新；候选先构建再原子写入；实际装填版本可见；提取、编辑、发射、更新、导入、导出和迁移保持低摩擦。

发射使用 `〔DCF·语言弹药〕` 标志，使接收窗口结合当前语境重新理解长期意图；清楚时适配后执行，关键冲突或无法可靠选择时才确认。更新使用 `〔DCF·弹药更新〕`，携带完整旧弹药并要求返回同 ID 完整 `DCF_AMMO`，不机械摘要当前对话。

Tampermonkey GM storage 不能被扩展直接读取。迁移通过旧正式版开放侧栏读取已经可见的真实弹药、发射模式和外观设置，进行数量、ID 与哈希校验；旧数据不删除，冲突不静默覆盖，不可观察的旧实现状态不猜测。

默认隐私边界是本地保存。除用户主动导出或检查官方代码更新，不上传对话或弹药，不扫描完整历史，不让诊断收集正文、令牌或认证信息。

一次实现迭代应形成一个原子业务提交、一次结构化 CI 读取和一个唯一候选产物。用户只进行一次正常使用验收，不承担插件故障模拟、控制台日志拼接或多版本切换。
