# DCF 基本共识提示词

本文件维护 DCF 项目的基础共识，用于新会话恢复方向感。

DCF 是用户与 AI 共同维护的个人 ChatGPT Tampermonkey 系统，不是企业插件平台或社区生态。它的价值目标是语言弹药闭环：从有价值的对话中提取、自动装填、更新并低摩擦发射可复用语言，使后续对话继续产生更好的认知材料。

语言弹药拥有价值主权，通用内核拥有工程主权。工程上先建立可扩展底座，语言弹药作为第一方核心模块运行其上；但不得为了通用性、纯洁性、安全形式或内部优化，取消、曲解或降低自动提取、自动装填、自动安装、自动更新和低摩擦发射。底座可以重构实现，不能重构目的；不能把内部复杂度重新交还给用户。

语言弹药是从过去语境沉淀出的高密度长期意图，不等于用户此刻逐字确认的即时命令。发射必须以轻量 DCF 调用标志触发当前语境的二次理解：清楚时适配后直接执行，关键冲突或多义时才确认。更新必须同时携带原弹药与修订协议，保持同一 `id`，吸收当前对话中已经稳定形成的变化，并返回完整可自动装填的 `DCF_AMMO` 工件。

DCF 的源码模块化，Tampermonkey 发布物仍是完整单文件。禁止远程 JavaScript 执行、eval、运行时分块引擎和 localStorage-as-code。

`dcf.state.root.v1` 是唯一权威状态。包、用户数据和必要系统元数据只通过统一事务改变；运行 registry、UI、回执和 catalog 状态都是派生或有界操作数据。事务统一执行候选变化、约束验证、投影、单根提交和回执，失败时旧根不变。

GM storage 可用时是唯一权威写入后端。page `localStorage` 只能作为旧版本迁移输入，不得与 GM storage 并列成双权威。切换存储 API 时必须显式桥接旧状态，不能把另一后端中的真实数据误判为不存在。

包 revision 不可变，外部领域格式编译成稳定资源地址。独占冲突必须在候选阶段失败，替换必须显式；用户结果与包定义分离，用户弹药、设置和外观不因可选包卸载而被反向删除。样式保留来源，不能拥有 `.sh` 几何；最终越界由实际可视区围栏处理。

安装包、运行模块、日常功能和维护工具是不同事实。包管理只表达安装与版本；日常功能承载主力工作流；探针、诊断、验收、作者、布局和恢复工具进入维护。`hidden` 不是产品角色，任何非弹药运行模块都必须在日常或维护分区保留可发现标题。界面密度由展开/折叠处理，折叠状态只属于可丢弃 UI session，不能进入权威根或改写模块包。

ChatGPT、GitHub catalog 和手动输入只是工件 transport。对话自动摄取只处理新完成的助手回复和固定恢复窗口，不扫描整页、不读取全部历史、不维护 DOM 块账本。相同工件由稳定身份自然幂等。DCF 对单条新回复的工作量不得随总对话轮数增长。

本地状态变化与外部副作用分离。composer、clipboard、notification 由 Host Adapter 执行；成功保持本地安静，失败形成可复制回执。证据不得改变被观察行为，且默认隐藏正文、提示词、内容、令牌和认证信息。

“一键 Runtime 体检并复制”只用于了解通过测试的脚本在用户当前浏览器标签页中实际运行成了什么。它必须独立观察实际存储、内存 Runtime、真实 Shadow DOM、host/observer/composer 和最近失败，只输出这些现场层之间无法合理解释的偏差。健康时保持 `deviations: []`；异常只附最小证据。体检不得把内部字段或与 UI 共用的推导函数直接提升为现场事实。

解决复杂问题时先做源头检查。来自日志、体检或 projection 的字段只能作为调查入口；解释前必须确认真实生产、消费和浏览器表现。当诊断与现场冲突时，诊断本身立即成为被调查对象。涉及语言弹药价值目标的部分，由源头化解题与价值约束共同求解；其他纯工程问题应改变事实模型、状态边界或统一机制，使复杂性失去产生条件。

DCF 的正常演化单位是不可变能力包 revision，而不是整份 userscript。完整对话包是按值交付，`DCF_PACKAGE_UPDATE` 是按引用交付；两者必须经同一个 Resolver/Reconciler 改变 `root.packages` 期望能力集合并重建 Runtime。普通功能、文案、布局和声明式 UI 优先作为包资源更新；userscript 只为启动、存储、校验、Host、协调器与恢复边界升级。

DCF 的统一架构对象是期望对话环境。`dcf.state.root.v1` 仍是唯一权威状态，Environment Snapshot 只是 Facade。对话工件、按钮和菜单先编译成同一种 typed intent；持久环境变化与一次性 Action/Effect 分离。包是不可变交付容器，content/action/view/style/policy 是环境资源。弹药、功能、构成和维护是同一环境的不同投影。Profile 和历史恢复都走环境迁移，不得另建工作区状态。
