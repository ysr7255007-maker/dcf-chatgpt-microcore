# DCF 基本共识提示词

本文件维护 DCF 项目的基础共识，用于新会话恢复方向感。

DCF 是用户与 AI 共同维护的个人 ChatGPT Tampermonkey 系统，不是企业插件平台或社区生态。它的价值目标是语言弹药闭环：从有价值的对话中提取、自动装填、更新并低摩擦发射可复用语言，使后续对话继续产生更好的认知材料。

语言弹药拥有价值主权，通用内核拥有工程主权。工程上先建立可扩展底座，语言弹药作为第一方核心模块运行其上；但不得为了通用性、纯洁性、安全形式或内部优化，取消、曲解或降低自动提取、自动装填、自动安装、自动更新和低摩擦发射。底座可以重构实现，不能重构目的；不能把内部复杂度重新交还给用户。

DCF 的源码模块化，Tampermonkey 发布物仍是完整单文件。禁止远程 JavaScript 执行、eval、运行时分块引擎和 localStorage-as-code。

`dcf.state.root.v1` 是唯一权威状态。包、用户数据和必要系统元数据只通过统一事务改变；运行 registry、UI、回执和 catalog 状态都是派生或有界操作数据。事务统一执行候选变化、约束验证、投影、单根提交和回执，失败时旧根不变。

GM storage 可用时是唯一权威写入后端。page `localStorage` 只能作为旧版本迁移输入，不得与 GM storage 并列成双权威。切换存储 API 时必须显式桥接旧状态，不能把另一后端中的真实数据误判为不存在。

包 revision 不可变，外部领域格式编译成稳定资源地址。独占冲突必须在候选阶段失败，替换必须显式；用户结果与包定义分离，用户弹药、设置和外观不因可选包卸载而被反向删除。样式保留来源，不能拥有 `.sh` 几何；最终越界由实际可视区围栏处理。

ChatGPT、GitHub catalog 和手动输入只是工件 transport。对话自动摄取只处理新完成的助手回复和固定恢复窗口，不扫描整页、不读取全部历史、不维护 DOM 块账本。相同工件由稳定身份自然幂等。DCF 对单条新回复的工作量不得随总对话轮数增长。

本地状态变化与外部副作用分离。composer、clipboard、notification 由 Host Adapter 执行；成功保持本地安静，失败形成可复制回执。证据不得改变被观察行为，且默认隐藏正文、提示词、内容、令牌和认证信息。

“一键体检并复制”是现场判断 DCF 全局状态的标准入口。它应足以核对存储后端、迁移覆盖、权威根、投影、包、模块、Surface、宿主监听和失败回执，同时不输出对话正文、弹药正文、包 payload、命令参数或认证数据。

解决复杂问题时先做源头检查。涉及语言弹药价值目标的部分，由源头化解题与价值约束共同求解；不直接牵涉价值目标的纯工程问题，应彻底改变事实模型、状态边界或统一机制，使复杂性失去产生条件，而不是继续堆分支和补丁。
