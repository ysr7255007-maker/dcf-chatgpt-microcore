# DCF 基本共识提示词

DCF 是用户与 AI 共同维护的个人认知基础设施。它必须替用户吸收复杂度，减少认知和操作负担；任何把安装、复制、确认、依赖判断、版本选择或维护工作重新交给用户的变化都是回退。

当前形态是一个只面向 Google Chrome 与 ChatGPT 的 Manifest V3 扩展。用户只安装一个载体。内部功能独立，外部仍是一个完整 DCF。

底座是无普通业务的生存载体，只理解独立插件工件、哈希、精确组合、注册、启动证据、更新、通用数据命名空间和恢复。统一侧栏、语言弹药、长对话减负、问答性能归因、外观、备份、功能管理和诊断都是普通独立插件。

每个插件是自足构建后 JavaScript，拥有稳定 ID、不可变版本、SHA-256 和独立 USER_SCRIPT world。浏览器不运行 npm、不解析依赖树、不使用 eval、Blob 或页面 CSP 绕行。

候选只有在全部启用插件返回匹配启动证据后才成为 current 与 last-known-good。失败自动恢复，不要求用户模拟故障或手工切换版本。恢复能力不得依赖动态插件。

功能插件从固定 GitHub 个人索引自动更新；Chrome 底座由 GitHub Actions 构建并通过非公开 Chrome Web Store 自动分发。正常成功保持安静，异常只向用户说明影响和已经采取的恢复结果，详细证据交给 AI。

产品功能参照 Core Review 前的完整 DCF Next。Next Core 是失败证据，不是架构来源。数据只接续 DCF Next 与 Chrome rc.1，不再单独迁移 `0.18.2`。
